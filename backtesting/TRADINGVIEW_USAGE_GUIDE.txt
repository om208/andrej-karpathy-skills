DETAILED SETTINGS EXPLANATION
================================================================================

SECTION 1: BASIC SETTINGS
─────────────────────────

🎯 Initial Capital ($)
  Default: 100
  Purpose: Starting capital for position sizing
  How to adjust: Increase/decrease based on account size
  Impact: Higher capital = larger positions per trade

🎯 Risk Per Trade ($)
  Default: 7
  Purpose: Maximum loss you're willing to take per trade
  How to adjust: Use 5-10% of account (e.g., $5-$10 for $100 account)
  Impact: Controls position size

🎯 Use Minimum Lot Sizes
  Default: TRUE
  Purpose: Uses smallest possible lot sizes
  How to adjust: Keep TRUE for safer positions
  Impact: Reduces per-trade risk

🎯 Lot Size (% of capital)
  Default: 0.35 (35%)
  Purpose: Size of each lot relative to capital
  How to adjust: 0.1 to 1.0 (0.1 = 10% each lot)
  Impact: Higher = bigger positions, higher risk

SECTION 2: SMA INDICATOR SETTINGS
──────────────────────────────────

📊 Enable SMA(196)
  Default: TRUE
  Purpose: Activates SMA calculation for touch detection
  How to adjust: Leave TRUE (required for strategy)
  Impact: If FALSE, strategy cannot detect signals

📊 SMA Period
  Default: 196
  Purpose: Simple Moving Average of 196 periods
  How to adjust: Keep at 196 for this strategy (don't change)
  Impact: Different periods = different trading signals
  Note: 196 period is validated on backtests = don't change

📊 SMA Touch Threshold (%)
  Default: 2.0%
  Purpose: How close SMA must be to candle high/low to count as "touch"
  How to adjust: 0.5% to 5.0%
  Impact:
    - Lower (0.5%) = stricter, fewer signals
    - Higher (5.0%) = looser, more signals
  Recommendation: Keep at 2% (validated)

📊 Show SMA on Chart
  Default: TRUE
  Purpose: Displays SMA line on your chart
  How to adjust: TRUE (see indicator) or FALSE (cleaner chart)
  Impact: Visual only, doesn't affect trades

SECTION 3: INSIDE BAR SETTINGS
───────────────────────────────

📐 Enable Inside Bar Detection
  Default: TRUE
  Purpose: Activates inside bar pattern detection
  How to adjust: Leave TRUE (required for strategy)
  Impact: If FALSE, strategy cannot detect signals

📐 Min Compression Ratio
  Default: 0.0
  Purpose: Minimum acceptable compression (0 = no minimum)
  How to adjust: 0.0 to 1.0
  Impact:
    - 0.35+ = filters out very loose compression
    - Keep at 0.0 for no minimum filter

📐 Max Compression Ratio
  Default: 1.0
  Purpose: Maximum acceptable compression (1.0 = no limit)
  How to adjust: 0.0 to 1.0
  Impact:
    - 0.65 = blocks medium compression (good for safety)

💰 Enable Lot 2
  Default: TRUE
  Purpose: Turns on Lot 2 (50% of position at 159 minutes)
  How to adjust: TRUE (enable) or FALSE (disable)

WHAT TO EXPECT: DETAILED BEHAVIOR
================================================================================

SIGNAL DETECTION FLOW
─────────────────────

1. CANDLE CLOSES
   → Every minute (on 1-min chart), a new candle forms
   → At candle close, strategy analyzes

2. INSIDE BAR CHECK
   → Is current_high < previous_high?
   → Is current_low > previous_low?
   → If YES to both → Inside bar found

3. SMA TOUCH CHECK
   → Is SMA(196) within 2% of candle range?
   → If YES → SMA touches inside bar

4. SIGNAL CONFIRMED
   → If both inside bar AND SMA touch → SIGNAL DETECTED
   → Now calculate risk score

5. RISK SCORE CALCULATION
   → Compression 0.20-0.35? Add 1 point
   → Compression 0.50-0.65? Add 1 point
   → Downward candle pattern? Add 1 point
   → Total score: 0, 1, or 2+

6. ENTRY DECISION
   → If risk score = 0 → ENTRY TAKEN
   → If risk score >= 1 → ENTRY SKIPPED (too risky)

POSITION MANAGEMENT FLOW
────────────────────────

AFTER ENTRY:
1. Lot 1 is active: Tracking +250 pip target OR 159 minutes
2. Lot 2 is active: Tracking 159 minutes only (no profit target)
3. Every candle, check exit conditions

LOT 1 EXIT:
- Condition A: High reaches entry + 250 pips
  → Exit Lot 1 at entry + 250 pips
  → Lock in profit
  → Keep Lot 2 open

- Condition B: 159 minutes elapsed
  → Exit Lot 1 at current close (whatever price)
  → May be profit or loss
  → Keep Lot 2 open

LOT 2 EXIT:
- Condition: 159 minutes elapsed
  → Exit Lot 2 at current close
  → Total position now closed
  → Ready for next signal

TOTAL P&L CALCULATION:
- P&L = (Lot 1 P&L) + (Lot 2 P&L)
- Win if total P&L > 0
- Loss if total P&L <= 0

RE-ENTRY:
- After both lots close, can enter immediately on next signal
- Must have new inside bar + SMA touch
- Must have risk score = 0

COMMON ISSUES & SOLUTIONS
================================================================================

ISSUE #1: NO SIGNALS APPEARING
─────────────────────────────

Problem: No trades are being taken
Possible causes:
  ✗ Enable SMA(196) is OFF
  ✗ Enable Inside Bar Detection is OFF
  ✗ Enable Risk Filtering is OFF but risk score > max allowed
  ✗ Timeframe not suitable (try 1-minute, 5-minute, or 15-minute)
  ✗ Market is consolidating (no inside bars forming)

Solution:
  1. Check: "Enable SMA(196)" = TRUE
  2. Check: "Enable Inside Bar Detection" = TRUE
  3. Check: "Enable Risk Filtering" = TRUE
  4. Check: "Max Acceptable Risk Score" = 0
  5. Try different timeframes (1-min is most reliable)
  6. Look for active markets (during trading hours)

ISSUE #2: TOO MANY FALSE SIGNALS
──────────────────────────────────

Problem: Lots of signals but most result in losses
Possible causes:
  ✗ Risk Filtering disabled (not using 85% system)
  ✗ Max Acceptable Risk Score too high (1 or 2)
  ✗ SMA Period wrong (not 196)
  ✗ SMA Touch Threshold too loose (>2%)

Solution:
  1. Enable Risk Filtering = TRUE
  2. Max Acceptable Risk Score = 0
  3. SMA Period = 196 (don't change)
  4. SMA Touch Threshold = 2.0% (don't change)
  5. Filter Very Tight = TRUE
  6. Filter Medium = TRUE

ISSUE #3: POSITIONS EXITING TOO EARLY
──────────────────────────────────────

Problem: Lot 1 exits quickly, Lot 2 still open, results in small losses
Possible causes:
  ✗ 250 pip target is too low (hitting quickly)
  ✗ 159 minutes is too short for volatility
  ✗ Position sizing too large (getting stopped out)

Solution:
  1. Increase Lot 1 TP to 350+ pips (test)
  2. Increase Lot 2 Hold Time to 200+ minutes (test)
  3. Reduce Lot Size % to 0.25 (smaller positions)
  4. Backtest to verify changes improve results

ISSUE #4: NOT REACHING 85% WIN RATE
────────────────────────────────────

Problem: Backtest shows 60-70% win rate instead of 85%
Possible causes:
  ✗ Wrong settings used (not matching validated parameters)
  ✗ Different market/timeframe (strategy optimized for BTC 1-min)
  ✗ Different time period (strategy has varying performance)

Solution:
  1. Use EXACTLY these settings:
     - SMA Period = 196
     - SMA Threshold = 2.0%
     - TP = 250 pips
     - Hold Time = 159 minutes
     - Risk Score Max = 0
  2. Use 1-minute chart
  3. Use BTC/USD (where it was validated)
  4. Run backtest over 500+ trades to get statistical significance
  5. Different markets may have different win rates
